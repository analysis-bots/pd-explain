352
353        :param right: Object to merge with.
354        :param how: Type of merge to be performed.
355                    * left: use only keys from left frame, similar to a SQL left outer join; preserve key order.
356                    * right: use only keys from right frame, similar to a SQL right outer join; preserve key order.
357                    * outer: use union of keys from both frames, similar to a SQL full outer join; sort keys lexicographically.
358                    * inner: use intersection of keys from both frames, similar to a SQL inner join; preserve the order of the left keys.
359                    * cross: creates the cartesian product from both frames, preserves the order of the left keys.
360        :param on: Column or index level names to join on. These must be found in both DataFrames.

375        :param suffixes: A length-2 sequence where each element is optionally a string indicating the suffix
376                         to add to overlapping column names in left and right respectively.
377                         Pass a value of None instead of a string to indicate that the column name from left or right
378                          should be left as-is, with no suffix. At least one of the values must not be None.
379
380

382        :param indicator: If True, adds a column to the output DataFrame called _merge with information on the source
383                          of each row. The column can be given a different name by providing a string argument.
384                          The column will have a Categorical type with the value of left_only for observations whose
385                          merge key only appears in the left DataFrame, right_only for observations whose merge key
386                          only appears in the right DataFrame, and both if the observations merge key is found in
387                          both DataFrames.
388        :param validate: If specified, checks if merge is of specified type.
389                         * one_to_one or 1:1: check if merge keys are unique in both left and right datasets.
390                         * one_to_many or 1:m: check if merge keys are unique in left dataset.
391                         * many_to_one or m:1: check if merge keys are unique in right dataset.
392                         * many_to_many or m:m: allowed, but does not result in checks.
